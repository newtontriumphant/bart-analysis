# bart-analysis
Hi there! Thanks for reviewing my project!

**NOTE: THIS IS A TRANSITTIME ENTRY**! :3

This was all done in one day (yes, like 6 hours of hackatime in 10 hours of day, I know I'm insane...)

This is a BART (Bay Area Rapid Transit) Analysis (Economics Extra Credit) using realtime API and GTFS data from BART, Wiener Linien, and the Munich Verkehrsbund.

Submission for HCTG TransitTime, double dipping with my school Economics Public Good project for extra credit because whyyy not (:

AI was used for about ~10% of the project code, and I tried to clearly denote in the code when AI was used. Most notably, when creating the function that interpolates BART data after pulling it was created by AI, as well as some CSS and random debug on the canvas. The rest is either my work or was found on stackoverflow or leaflet docs (obviously not counting inline completions)!

bart-data.py is a file that outputs correct parsed values of all BART stations (I will definitely reuse this in the future!)

api/proxy.js is a replacement for corsproxy.io since vercel is stupid and blocks it.
